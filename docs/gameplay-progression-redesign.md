# 日常 / 科研 / 团队 · 进展与行动体系重构设想

> 文档性质：设计草案，供策划评审与迭代；未绑定具体实现排期。  
> 背景：当前大量行动在数值上可互相替代，速通玩家可压缩为「少数高性价比点击」，削弱差异化与博士生培养过程的叙事真实感。

---

## 1. 问题归纳

### 1.1 现象

- **Fungible progress（可替代进度）**：多数行动同时贡献「阶段进度 + 声望/好感 + 资源结构微调」，缺少「非做不可」或「做了 A 才能解锁 B」的硬结构。
- **最优解趋同**：在固定每季行动上限下，玩家很快识别出少数「进度/风险/成本」比最优的行动，其余沦为 flavor click。
- **科研链扁平**：文献 / 实验 / 撰写 / 出差在叙事上有区别，但在通关路径上常被简化为「堆撰写 + 审稿 RNG」。
- **日常与团队边缘化**：修课、讲坛、助教、团建、加班、拜访等，对**通关硬门槛**的约束力弱，易被跳过。

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **差异化** | 不同行动在至少一个维度上「不可替代」或「显著更优」，避免纯数值换皮。 |
| **阶段感** | 与真实博士培养阶段（课程 → 入组 → 开题 → 中期 → 论文 → 答辩）对齐，各阶段**主矛盾**不同。 |
| **可读策略** | 玩家能说出「我这季度为什么要点这个」，而不是「随便点满 20 次」。 |
| **保留诙谐基调** | 规则可以硬，文案仍保持中关村学院式黑色幽默，避免变成功课表模拟器。 |

---

## 2. 总体思路：三条轴 + 强制交织

建议把进展从「单条 progress 条」拆成玩家可理解的 **三条资源轴**（名称可再包装）：

1. **培养合规轴**（教务/学分/培养环节节点）— 对应 **日常** 为主。  
2. **科研产出轴**（想法 → 证据 → 文本 → 传播）— 对应 **科研** 为主。  
3. **课题组生态轴**（导师关系 + 同门网络）— 对应 **团队** 为主。

**强制交织**：关键门槛（开题、中期、答辩前检查）不要求三条都满，但要求 **在窗口期内完成过指定类型的行动次数或里程碑**，避免单轴刷穿。

---

## 3. 分阶段：行动如何「必须有用」

以下用「机制方向」表述，具体数值需单独平衡表。

### 3.1 课程学习 / 导师选择

- **修读课程**：不仅给学分，可设为 **阶段进度的主要合法来源之一**（例如：未达某学分前，纯科研推进度效率衰减），避免「一天课不上只肝实验」的违和速通。  
- **校园与讲坛**：绑定 **声望软门槛** 或 **随机正面事件权重**（例如：中期前至少 N 次讲坛，降低「答辩风评」类隐性判定难度）。  
- **担任助教**：与 **资金** 强绑定已较好；可增设 **与导师好感小幅联动** 或 **减轻下一季某类科研理智消耗**（叙事：助教杂活换组里人情）。

### 3.2 开题准备 ~ 开题报告

- **文献调研**：设为 **论文撰写进度 / 投稿前置条件** 的组成部分，例如：  
  - 「文献积累层数」不足时，撰写进度上限锁在 80%；或  
  - 每完成一篇投稿前，本季度需至少 1 次文献（软强制）。  
- **运行实验**：与 **算力** 已绑定；可增设 **实验记录点数**，影响 **审稿接受率或 misconduct 风险**（叙事：没实验硬写 = 风险）。  
- **论文撰写**：保持高压核心；与 **实验/文献** 的耦合见上。  
- **出差报告**：定位为 **声望爆发 + 降低长期 reputation 瓶颈**；可要求 **至少 1 篇已发表或投稿在审** 才允许（避免开局刷会）。

### 3.3 中期考核 ~ 论文撰写

- **集体加班 / 团建 / 拜访导师**：从「可选好感」升级为 **课题组压力计**：  
  - 例如：**连续两季无导师互动** → 下季科研理智消耗 +X% 或 advisorFavor 衰减；  
  - **同门好感过低** → 某类随机事件（数据丢失、借不到卡）概率上升。  
- 目的不是惩罚休闲玩家，而是让 **团队行动成为风险对冲**，而非纯装饰。

### 3.4 毕业答辩前

- **硬门槛已有**（论文数、声望）：可追加 **「培养档案完整性」** 软检查：  
  - 例如：全学制内 **日常 / 科研 / 团队** 各类行动至少完成过下限次数，否则答辩对话进入「刁钻模式」（提高失败叙事概率或增加额外季度），**不直接判死**，但增加戏剧张力。

## 4. 让行动「有特色」的几种可组合手段

任选其几组合，不必一次全上。

### 4.1 前置与互斥（Prerequisite）

- 示例：`write_paper` 要求本季度或上季度有过 `literature_review` 或 `run_experiments`。  
- 示例：`conference_trip` 要求 `papersPublished ≥ 1` 或 `submittedPapers ≥ 1`。

### 4.2 层数 / 充能（Stack / Charge）

- 文献层数、实验记录层数、组会存在感——影响 **撰写效率、审稿、事件池**，而不是只加 6% progress。

### 4.3 效率窗口（Seasonal Modifier）

- 每季随机 1～2 个「学院重点」标签（如：本学期强调交叉应用）→ 对应行动 **进度系数 ×1.25**，其它 **×0.9**，迫使阵容轮换。

### 4.4 冷却与配额（Cooldown / Category Cap）

- 同类行动连点收益递减（防止 20 次全写同一按钮）。  
- 或对 **DAILY / RESEARCH / TEAM** 设 **每季每类最低次数**（完成有小幅奖励，不完成有轻微 debuff）。

### 4.5 叙事绑定（Story Flags）

- 关键剧情节点只在与 **特定行动** 组合时触发（例如：只在「讲坛」后触发某大牛邮件事件）。

---

## 5. 与现有系统的关系

- **季度行动上限**、**精力/算力/资金** 可保留；改动重点在 **「什么行动在什么阶段值得点」** 的规则层。  
- **引用 / 论文 / 荣誉** 等已有子系统可作为 **科研轴** 的展示层，不必推倒重来。  
- **自我调节**（散步、摸鱼等）建议保持 **弱绑定**，避免压力过大；最多与 **团队 debuff** 轻量联动。

---

## 6. 实施建议（分阶段）

| 阶段 | 内容 | 风险 |
|------|------|------|
| **P0** | 1～2 条硬前置（如撰写←文献/实验、出差←有稿）+ UI 提示 | 低，易理解 |
| **P1** | 引入 1 个可数值化层数（如文献层）影响撰写或审稿 | 中，需调表 |
| **P2** | 导师/同门「长期冷落」衰减与团队行动对冲 | 中，需防误伤新手 |
| **P3** | 季度学院焦点、行动分类配额、答辩档案检查 | 高，需大量测试 |

---

## 7. 待你拍板的问题

1. **硬核程度**：更偏向「规则强制」还是「软激励 + 叙事惩罚」？  
2. **单局时长**：是否接受「同样通关季度数略增」换取决策密度？  
3. **随机性**：审稿、事件已较随机；行动结构是否应 **再降 RNG**，更多靠规划？  
4. **多周目**：是否需要为老玩家提供「挑战模式」规则集，与首周目分流？

---

## 8. 小结

核心主张是：**减少「任意行动 = 进度条 +N」的同质化**，用 **阶段主矛盾 + 科研链条 + 课题组生态** 三条轴交叉约束，让日常 / 科研 / 团队在各自阶段都有「不点会亏、点了有故事」的理由；实现上可渐进落地，先前置与耦合，再层数与赛季修饰。

若你认可大方向，下一步可拆成：**具体数值表（每行动每阶段的权重）** + **玩家可见的 UI 提示（为何此刻推荐修课/跑实验）** 两份子文档，再进入开发排期。
